​ Git是目前最流行的分布式版本控制系统之一。它被广泛用于管理软件和其他文本文件的版本控制，并协助多人协同开发。Git由Linux操作系统的内核创始人Linus Torvalds在2005年创造，以处理其自己的内部版本管理需求。自那时起，Git已经成为一个重要的工具，被许多公司和开源项目用来维护代码库、跟踪更改并记录历史。

3. 工作区域和文件状态

git的本地数据管理分为三个区域：

  1. 工作区（Working Directory）：实际操作的目录。
  2. 暂存区（Staging Area/Index）：中间区域，用于临时存放即将提交的修改内容。
  3. 本地仓库（Local Repository）：Git存储代码和版本信息的主要位置。

git的文件存在四种状态：

  1. 未跟踪（Untrack）：新创建的还未被Git管理起来的文件。
  2. 未修改（Unmodified）：已经被Git管理起来但文件的内容没有发生变化的文件。
  3. 已修改（Modified）：已经修改但还没有添加到暂存区的文件。
  4. 已暂存（Staged）：已经修改也已经添加到暂存区的文件。

8. 忽略文件

应该忽略以下文件：

  • 系统或者软件自动生成的文件
  • 编译产生的中间文件和结果文件
  • 运行时生成日志文件、缓存文件、临时文件
  • 涉及身份、密码、口令、密钥等敏感信息文件

.gitignore文件的匹配规则

从上到下逐行匹配，每一行表示一个忽略模式

  • 空行或者以#开头的行会被Git忽略。一般空行用于可读性的分隔，#一般用作注释
  • 使用标准的Blob模式匹配，例如：
    • *通配任意个字符
    • ?匹配单个字符
    • []表示匹配列表中的单个字符，比如：[abc] 表示 a/b/c
  • 两个 ** 表示匹配任意的中间目录
  • 中括号可以使用短中线连接，比如：[0-9]
  • !表示取反，忽略指令模式以外的文件或目录
